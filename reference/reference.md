下面是这个项目的补充参考资料，请你在实现 Stylish 时严格参考这些资料，但不要直接照搬品牌视觉，而是提炼“设计语言特征”，做成 inspired-by 的三套主题。

项目名称：
Stylish

项目目标：
构建一个可部署到 Vercel 的静态展示网站，展示同一套页面结构在三种设计语言下的差异：
- Apple-like
- Google-like
- Microsoft-like

请注意：
- 不要直接复制 Apple、Google、Microsoft 的官方页面
- 不要使用它们的 logo、商标、品牌专有素材
- 目标是提炼风格特征，形成三套抽象主题
- 三套主题必须共用同一套组件架构，而不是复制三份页面

====================
一、总体设计要求
====================

我希望你重点参考以下维度：
- 留白节奏
- 圆角体系
- 阴影轻重
- 边框存在感
- 卡片层级
- 导航方式
- 字体层级倾向
- 信息密度
- 表单和按钮气质
- 页面整体的“产品感 / 工具感 / 亲和感”

请你把这些差异沉淀为可切换的 theme tokens，而不是写死在页面里。

====================
二、Apple-like 参考
====================

目标感觉：
- 极简
- 克制
- 高级
- 大留白
- 更安静
- 更适合产品展示页
- 页面层级更依赖留白和排版，而不是重边框

- 更精致的标题与正文比例

====================

====================

目标感觉：
- 友好
- 清晰
- 更有活力
- 色彩更丰富但仍然受控
- 更圆润
- 更强调组件状态
- 更适合面向大众用户的 Web App

网页参考：

- [例如：更明显的状态色和交互反馈]
- [例如：卡片、输入框、按钮更圆润]

请提炼出的设计特征：
- 主色更明显
- 圆角更柔和
- 交互状态更可见
- 模块之间更有亲和力
- 控件更偏 approachable
- 页面层次清楚，但不要太厚重

====================
四、Microsoft-like 参考
====================

目标感觉：
- 生产力
- 结构化
- 更像后台 / 控制台 / 仪表盘
- 分区明确
- 信息密度略高
- 导航更清晰
- 更适合 OA / Admin Dashboard

- [例如：导航、表格、设置区块的工具感]

请提炼出的设计特征：
- 边界更明确

- 面板感更明显

- 内容区块更规整
- 密度略高但保持现代感

====================
五、如果我还提供 CSS / Design Tokens
====================

如果我提供 CSS 文件、Tailwind 配置或 token 文件，请这样处理：
- 优先把它们理解为“实现参考”
- 不要机械照抄每条样式
- 把它们整合进统一的主题系统
- 尽量沉淀为可维护的 token 结构
- 保持组件层与主题层分离

====================
六、实现优先级
====================

请优先保证以下几点：
1. 主题切换架构正确
2. 三个场景共用同一套组件
3. 三种主题差异明显，不只是换颜色
4. 首页和三个场景页完整可展示
5. 代码结构清晰，适合放到 GitHub
6. 可直接部署到 Vercel
